# AGENTS.md

## Purpose

This repository follows the AI Assisted Engineering Framework v1.0.

The objective is not rapid code generation.

The objective is to produce:

* secure solutions
* reliable solutions
* observable solutions
* maintainable solutions
* cost-efficient solutions
* operationally sustainable solutions
* governed solutions
* validated solutions

Always optimize for engineering outcomes rather than implementation speed.

---

## Context Loading Order

Before making significant recommendations or changes:

1. Read AGENTS.md
2. Read METADATA.md
3. Read Context Pack (01-context/context-pack.md)
4. Read Requirements Specification (02-requirements/)
5. Read Architecture Blueprint (04-architecture/)
6. Read ADRs (04-architecture/adr/)
7. Read Validation Artifacts (06-validation/)
8. Read Operational Artifacts (07-operations/)

If required information is missing:

* identify gaps
* document assumptions
* request clarification

Avoid proceeding with critical assumptions.

---

## Required Engineering Process

1. Understand Context
2. Understand Requirements
3. Identify Risks
4. Evaluate Governance Implications
5. Design Solution
6. Define Validation Approach
7. Implement
8. Validate
9. Prepare Operational Readiness
10. Capture Reusable Intelligence

Do not jump directly into implementation.

---

## Engineering Priorities

When trade-offs exist:

1. Security
2. Governance
3. Reliability
4. Availability
5. Observability
6. Optimization
7. Sustainability
8. Performance
9. Convenience

---

## AI Participation

AI may: analyze, summarize, explain, recommend, draft, review, optimize, identify risks and gaps.

AI must not: approve decisions, bypass governance, override validation, deploy uncontrolled changes, disable security controls, make irreversible decisions without review.

Human accountability always takes precedence.

---

## Definition of Done

A solution is not complete until:

* requirements are satisfied
* architecture is aligned
* security is reviewed
* observability is considered
* operational impact is reviewed
* validation is completed
* risks are identified
* trade-offs are documented

Working code alone does not constitute completion.
